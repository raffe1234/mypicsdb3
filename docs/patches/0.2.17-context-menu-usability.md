# Patch report: 0.2.17 context-menu usability

## Scope

Version 0.2.17 makes the existing minimum-rating policy easier to reach and
removes a duplicate album context-menu action. Database schema 2 and all
migration checksums remain unchanged.

Included:

- **Minimum picture rating** in Kodi's context menu throughout MyPicsDB 3;
- the existing add-on settings dialog as the single place where the policy is
  changed;
- removal of the per-item album-view action that duplicated the globally
  registered Kodi context item;
- XML registration tests and Kodi UI smoke coverage.

Excluded:

- rating-policy semantics or SQL changes;
- scanner, metadata or database changes;
- Estuary skin changes.

## Manual Kodi checks before release tagging

1. Update MyPicsDB 3 to 0.2.17. The Estuary MyPicsDB 3 skin may remain 21.3.5.
2. Press **C** on a picture, album and main-menu entry and confirm **Minimum
   picture rating** is available.
3. Select it and confirm the MyPicsDB 3 settings dialog opens on the General
   category.
4. Set the policy to **Rating 3 or higher**, return to the add-on main menu and
   confirm **Minimum rating: 3+** and **Show all pictures temporarily** appear.
5. Open an album, press **C** and confirm **Save current view as album default**
   appears only once.
6. Confirm the selected policy still filters pictures, counts and representative
   artwork as documented for 0.2.16.
