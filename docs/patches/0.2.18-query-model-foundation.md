# Patch report: 0.2.18 Query Model foundation

## Scope

Version 0.2.18 adds the internal Query Model foundation planned before global
search, smart filters, saved views and smart collections.

The patch adds:

- Query Model version 1 immutable Python objects;
- strict parsing and normalization of untrusted JSON-compatible dictionaries;
- nested all/any groups and group negation;
- bounded rules, nesting, list sizes and strings;
- allowlisted rating, favorite, source, album, capture-date, camera and keyword
  rules;
- allowlisted stable sorting with an ID tie-breaker;
- deterministic canonical JSON;
- backend-neutral SQL compilation using bound parameters only;
- catalogue page and count methods using the same compiled predicate;
- optional application of the local minimum-rating policy;
- SQLite execution and validation tests;
- an opt-in MySQL/MariaDB parity test.

## Compatibility

- Database schema remains version 2.
- Existing browser routes, widgets, scans and stored metadata are unchanged.
- No query data is stored in the database.
- No new Kodi menu or settings entry is added.
- Estuary MyPicsDB 3 remains independently versioned.

## Security boundary

The parser rejects unknown members, fields, operators, sorts and model versions.
No user value becomes part of an SQL string. Keyword, camera and other string
values are carried only in the parameter tuple.

## Verification

Run:

```bash
python3 -m pytest -ra
python3 tools/verify.py
python3 tools/build.py --skip-skin
git diff --check
```

The MySQL/MariaDB workflow additionally runs the opt-in integration test and
verifies page/count parity plus minimum-rating-policy behaviour.

## Follow-up

The next planned functional patch is global search built on Query Model version
1. Search tokenization and indexing should remain separate from this foundation
until realistic SQLite/NAS and MariaDB measurements are available.
