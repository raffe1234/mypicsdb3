# Patch report: 0.2.13 migration foundation

## Purpose

Provide the first incremental, future-safe patch from the project handover
without implementing user-facing phase-2 features.

## Included

- versioned migration runner and schema history;
- schema-1 baseline registration;
- SQLite WAL checkpoint, free-space check, atomic backup, and integrity check;
- MySQL/MariaDB server preflight and backup warning;
- refusal of newer database schemas before structural writes;
- migration/checksum validation and rollback tests;
- shared scan/migration lock policy;
- read-only current and legacy schema inspection tools;
- migration ADR, operating documentation, and release notes.

## Deliberately excluded

- schema version 2 tables;
- global search and Query Model;
- smart filters and collections;
- generic metadata facets;
- source include/exclude rules;
- export and ZIP;
- legacy database import;
- skin changes.

## Validation

Run before merging:

```bash
git diff --check
python3 -m pytest -q
python3 tools/verify.py
```

Optional MySQL/MariaDB integration test:

```bash
MYPICSDB3_TEST_MYSQL=1 python3 -m pytest tests/test_mysql_integration.py
```

## Recommended next patch

Add the first real schema-2 migration as a small isolated change. Prefer date
indexes and the global rating-policy foundation before saved views or search.
The new migration must include upgrade fixtures for SQLite and an opt-in
MySQL/MariaDB integration path.
