# Patch report: 0.2.15 schema-2 date browsing

## Purpose

Record the first released schema migration and distinguish automated coverage
from the manual Kodi validation that is still required.

## Included

- schema version 2 with `idx_pictures_date_browse` for SQLite and
  MySQL/MariaDB;
- year → month → day → pictures navigation;
- a separate **No date** folder;
- preserved route parameters for paginated date, camera and keyword views;
- fresh-database, schema-1 upgrade, rerun, Kodi UI and MariaDB tests;
- ADR 0002 documenting the deliberately narrow index-based design.

## Deliberately excluded

- global rating policy;
- common Query Model and global search;
- smart filters, saved views and collections;
- generic metadata facets;
- source include/exclude rules;
- export and ZIP;
- legacy database import.

## Automated validation

Run before merging or releasing:

```bash
git diff --check
python3 -m pytest -ra
python3 tools/verify.py
python3 tools/build.py --skip-skin
```

The opt-in MySQL/MariaDB integration test must also pass in CI.

## Manual validation still required

On a copy of a real schema-1 SQLite catalogue, with Kodi stopped before copying
or restoring database files:

1. Install or refresh to MyPicsDB 3 version 0.2.15.
2. Confirm that a verified backup is created under the add-on profile's
   `backups` directory.
3. Confirm `meta.schema_version = 2` and migration-history rows for versions 1
   and 2.
4. Compare source, picture and album counts before and after migration.
5. Restart Kodi and confirm that migration startup is idempotent.
6. Browse Years, months, days, pictures and **No date**, including pagination.
7. Recheck On this day, Cameras and Keywords pagination.
8. Stop Kodi, restore the backup and verify that the catalogue opens correctly.

For MySQL/MariaDB, use an externally backed-up test copy and repeat the schema
and browsing checks before production use.

## Recommended next functional patch

Add a global minimum-rating display policy without changing the database
schema. Keep counts and pagination under the same central predicate, make the
active policy visible, and preserve a clear way to show all pictures.
