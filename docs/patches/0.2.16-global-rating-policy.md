# Patch report: 0.2.16 global rating display policy

## Scope

Version 0.2.16 adds a global minimum-rating policy as a local display setting.
Database schema 2 and all released migration checksums remain unchanged.

Included:

- settings choices for all, rated and unrated, and thresholds 1 through 5;
- explicit `NULL` versus rating-0 semantics;
- one validated catalogue predicate shared by SQLite and MySQL/MariaDB;
- consistent filtering of picture rows, pagination, album counts and artwork,
  date groups, cameras, keywords and widgets;
- visible policy status and a temporary all-pictures browsing override;
- SQLite unit tests, Kodi UI tests and an opt-in MariaDB parity test.

Excluded:

- rating changes or metadata writeback;
- scanning or storage filters;
- Query Model, saved views, collections and search;
- any database migration.

## Policy semantics

| Setting | Included ratings |
| --- | --- |
| All pictures | `NULL`, 0, 1, 2, 3, 4, 5 |
| Rated and unrated | `NULL`, 1, 2, 3, 4, 5 |
| 1+ | 1, 2, 3, 4, 5 |
| 2+ | 2, 3, 4, 5 |
| 3+ | 3, 4, 5 |
| 4+ | 4, 5 |
| 5 | 5 |

`NULL` means that no embedded rating was found. Zero is a stored, explicit
rating and can therefore be excluded independently.

## Manual Kodi checks before release tagging

1. Update a test installation to 0.2.16 and confirm schema remains version 2.
2. Set **Minimum picture rating** to each available choice.
3. Check Recently taken, Recently added, albums, Years, Cameras, Keywords,
   Favorites, Rated pictures and one home-screen widget.
4. Confirm list contents, folder counts and folder artwork agree.
5. Open **Show all pictures temporarily**, navigate through an album and date
   hierarchy, and confirm the override remains active.
6. Return to **Use configured rating filter** and confirm the saved setting was
   not changed.
7. Run a scan and confirm lower-rated pictures remain stored and reappear when
   the policy is changed back to All pictures.
8. Repeat the main query checks against MariaDB when a shared backend is used.
